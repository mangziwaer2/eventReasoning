# 事件输入规范

## 1. 方法边界

本项目的方法从结构化事件开始：

```text
query + documents + pre-extracted events
-> candidate pairs
-> coarse graph
-> refinement
-> future event forecasting
```

事件抽取由数据集标注或冻结的外部模型完成，不参与本项目训练。其他使用者需要先把自己的事件抽取结果转换为 `event-input-v1`。

## 2. 必需字段

每个事件至少包含：

```json
{
  "event_id": "e1",
  "trigger": "attacked",
  "mention": "forces attacked the village",
  "document_id": "doc_001",
  "sentence_index": 4,
  "participants": ["forces", "village"],
  "confidence": 0.93,
  "evidence": "The forces attacked the village before dawn."
}
```

约束：

- `event_id` 在同一个 query 内唯一。
- `trigger` 和 `mention` 非空。
- `document_id` 必须属于当前 query 的历史文档。
- `sentence_index` 为非负整数。
- `confidence` 位于 `[0, 1]`。
- `evidence` 可为字符串、对象或对象数组。
- 不允许把 gold relation edges 写入事件输入作为模型特征。

内部节点文本统一为：

```text
trigger=attacked; mention=forces attacked the village
```

## 3. 自包含 JSON

自包含文件适合单文档或自定义数据推理，包含 query、documents 和 events。完整示例见：

```text
examples/event_input.example.json
```

校验：

```bash
python src/event_input.py --input examples/event_input.example.json
```

粗图推理：

```bash
python src/run_coarse_graph_qwen.py \
  --input-mode events \
  --input examples/event_input.example.json \
  --base-model-path models/Qwen2.5-0.5B \
  --adapter-path outputs/coarse_graph_qwen_lora_4090_full/best_adapter \
  --output outputs/example_coarse_graph.json
```

## 4. MIRAI JSONL

MIRAI 批量评测不需要在每行重复 query 和 documents。每行只需：

```json
{"schema_version":"event-input-v1","sample_id":"mirai_1","query_id":"1","events":[...]}
```

评测器根据 `query_id` 从 `MIRAI_data.zip` 加载权威 query 和 cutoff 前文档，然后检查每个事件的 `document_id`。事件 JSONL 中即使包含 query/documents，MIRAI 评测也不会用它们替换数据集上下文。

每个待评测 query 必须恰好有一条记录。缺失 query、重复 query、重复 event ID 或未知 document ID 会在模型加载前直接报错。

## 5. 两种论文设置

### Gold-event setting

- MAVEN-ERE 直接读取 gold event mentions。
- 用于训练和隔离评测图构建能力。
- 不代表真实部署中的抽取质量。

### Frozen-extractor setting

- 使用一个固定、公开说明版本的外部事件抽取器。
- 抽取器对 MIRAI 只运行一次并缓存 JSONL。
- 所有 documents-only、events-only、coarse、refined 和 shuffled 对照复用同一份事件文件。
- 抽取器参数不进入 optimizer，不根据 MIRAI test 结果修改。

论文需要报告抽取器名称、版本、prompt 或 checkpoint、生成时间和事件统计，但不需要把事件抽取作为创新点。

## 6. 防止信息泄漏

1. 只处理 cutoff 之前的文档。
2. 外部抽取器不能看到 MIRAI answer、future documents 或 relation label。
3. 事件文件只保存观察事件，不保存未来预测或 gold edges。
4. MIRAI query/documents 始终从数据集重新加载。
5. 保存 extractor 配置和事件文件哈希，保证所有方法对照使用相同输入。
