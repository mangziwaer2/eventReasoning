# 训练与评测操作手册

本文档只记录当前有效命令。研究目的、方法选择和论文计划见 [项目书](项目书.md)。

## 1. 目录约定

本地从仓库根目录执行：

```text
lite_causal_eventrag/
```

云端若进入 `src/` 目录，命令中的脚本不再带 `src/` 前缀。`cloud_training.txt` 当前按云端 `src/` 工作目录编写。

默认资源路径：

```text
datasets/MAVEN_ERE.zip
datasets/MIRAI_data.zip
datasets/mirai_events_test.jsonl    冻结外部抽取器预计算事件
models/Qwen2.5-0.5B
models/Qwen2.5-4B                 可选，供最终预测
outputs/
```

## 2. 环境检查

安装 Python 依赖：

```bash
pip install -r requirements.txt
```

PyTorch 应按服务器 CUDA 版本单独安装。运行前检查：

```bash
python -c "import torch; print(torch.__version__, torch.cuda.is_available(), torch.cuda.get_device_name(0))"
```

LoRA 检查：

```bash
python -c "import transformers, peft; print(transformers.__version__, peft.__version__)"
```

若日志出现 `torchao` 动态库加载失败，但项目没有使用 torchao 量化，优先卸载不兼容的 torchao；若需要 torchao，则升级到当前 peft 要求的兼容版本。

## 3. 数据检查

项目方法从预抽取事件开始。先校验标准输入样例：

```bash
python src/event_input.py \
  --input examples/event_input.example.json
```

MIRAI 批量事件文件使用 `event-input-v1` JSONL，每个 query id 恰好一条记录。格式与防泄漏要求见 [事件输入规范](事件输入规范.md)。

查看少量 MAVEN 事件对：

```bash
python src/coarse_graph_dataset.py \
  --split train \
  --limit 2 \
  --max-events 16 \
  --negative-ratio 1.5
```

查看少量 refinement 样本：

```bash
python src/refinement_dataset.py \
  --mode maven \
  --split train \
  --limit 2 \
  --max-events 16
```

检查事件节点时应看到具体 `trigger + mention`，而不是把任意完整句子直接当作事件。MAVEN 数据由数据集 gold annotations 提供，不运行项目内事件抽取器。

## 4. 粗图 Qwen LoRA

### 4.1 小规模 smoke test

```bash
python src/train_coarse_graph_qwen.py \
  --train-limit 32 \
  --validation-limit 8 \
  --max-events 12 \
  --max-train-pairs 512 \
  --max-validation-pairs 128 \
  --negative-ratio 1.0 \
  --epochs 1 \
  --batch-size 2 \
  --eval-batch-size 2 \
  --output-dir outputs/coarse_graph_qwen_smoke
```

### 4.2 4090 正式训练

```bash
python src/train_coarse_graph_qwen.py \
  --model-path models/Qwen2.5-0.5B \
  --train-limit 0 \
  --validation-limit 0 \
  --max-events 16 \
  --negative-ratio 1.5 \
  --max-sentence-gap 3 \
  --epochs 10 \
  --max-length 1024 \
  --batch-size 4 \
  --eval-batch-size 4 \
  --tokenize-batch-size 512 \
  --gradient-accumulation-steps 4 \
  --lr 2e-4 \
  --document-mode title \
  --log-every 25 \
  --debug-samples 2 \
  --output-dir outputs/coarse_graph_qwen_lora_4090_full
```

`--train-limit` 和 `--validation-limit` 限制的是 MAVEN 文档行数，不是展开后的事件对数。`0` 表示完整 split。一篇文档会展开成很多候选事件对，因此加载后样本数远大于 limit 是正常现象。

`--negative-ratio 1.5` 表示每个正关系事件对大约采样 1.5 个 `none` 负事件对。它控制训练类别与计算量，不是关系阈值。

### 4.3 续训

```bash
python src/train_coarse_graph_qwen.py \
  --model-path models/Qwen2.5-0.5B \
  --train-limit 0 \
  --validation-limit 0 \
  --max-events 16 \
  --negative-ratio 1.5 \
  --epochs 10 \
  --batch-size 4 \
  --eval-batch-size 4 \
  --gradient-accumulation-steps 4 \
  --resume-from outputs/coarse_graph_qwen_lora_4090_full \
  --output-dir outputs/coarse_graph_qwen_lora_4090_full
```

若存在 `latest_training_state.pt`，会恢复 optimizer、epoch、global step 和 history；否则从 `latest_adapter` 恢复模型并根据 history 推断起始 epoch。

### 4.4 输出

```text
train_config.json
train_history.json
training.log
debug_predictions.jsonl
debug_readable.log
best_adapter/
latest_adapter/
latest_training_state.pt           若训练版本已生成
```

正式评测使用 `best_adapter/`。`latest_adapter/` 只表示最后一次保存，不保证验证集最佳。

## 5. Refinement

### 5.1 小规模 smoke test

```bash
python src/train_refinement.py \
  --dataset-mode maven \
  --limit 64 \
  --max-events 12 \
  --epochs 2 \
  --hidden-dim 128 \
  --message-steps 3 \
  --output-dir outputs/refinement_smoke
```

### 5.2 4090 正式训练

```bash
python src/train_refinement.py \
  --dataset-mode maven \
  --limit 0 \
  --max-events 16 \
  --negative-completion-ratio 0.75 \
  --max-completion-edges 0 \
  --validation-ratio 0.1 \
  --epochs 40 \
  --hidden-dim 192 \
  --message-steps 4 \
  --dropout 0.12 \
  --lr 3e-4 \
  --weight-decay 1e-4 \
  --grad-clip 1.0 \
  --keep-loss-weight 1.0 \
  --type-loss-weight 0.7 \
  --strength-loss-weight 0.3 \
  --density-loss-weight 0.08 \
  --keep-pos-weight auto \
  --type-class-weighting auto \
  --amp auto \
  --log-every 25 \
  --debug-samples 2 \
  --output-dir outputs/refinement_graph_4090_full
```

当前完整 run 在约第 16 epoch 后出现 validation loss 回升。正式推理使用保存的 best `refinement_model.pt`，不要使用 `refinement_model_latest.pt`。

训练脚本尚未实现命令行 resume。重启 refinement 训练前应先保留原输出目录；后续代码任务会补 checkpoint resume 和 early stopping。

### 5.3 输出

```text
train_config.json
train_history.json
debug_predictions.jsonl
debug_readable.log
refinement_model.pt                最佳验证 checkpoint
refinement_model_latest.pt         最后一个 epoch
best_training_state.pt
latest_training_state.pt
```

`debug_readable.log` 中：

- `source=coarse`：候选来自粗图已有边。
- `source=completion`：候选来自非粗图事件对补集的筛选结果。
- `coarse=precedes:0.390`：候选的初始关系先验与分数，不表示粗图一定保留了该边。
- `gold=KEEP:precedes`：仅训练/验证日志可见的监督标签，实际推理没有 gold。

## 6. 单图推理

从预抽取事件生成 coarse graph：

```bash
python src/run_coarse_graph_qwen.py \
  --input-mode events \
  --input examples/event_input.example.json \
  --base-model-path models/Qwen2.5-0.5B \
  --adapter-path outputs/coarse_graph_qwen_lora_4090_full/best_adapter \
  --max-events 16 \
  --max-pairs 64 \
  --keep-threshold 0.5 \
  --output outputs/coarse_query_1.json
```

生成 refined graph：

```bash
python src/run_refinement.py \
  --coarse-graph outputs/coarse_query_1.json \
  --model-path outputs/refinement_graph_4090_full/refinement_model.pt \
  --include-completion-candidates \
  --max-completion-edges 64 \
  --keep-threshold 0.5 \
  --output outputs/refined_query_1.json
```

## 7. MAVEN 图评测

先用小样本确认模型和路径：

```bash
python src/evaluate_maven_pipeline.py \
  --split test \
  --limit 16 \
  --base-model-path models/Qwen2.5-0.5B \
  --coarse-adapter-path outputs/coarse_graph_qwen_lora_4090_full/best_adapter \
  --refinement-model-path outputs/refinement_graph_4090_full/refinement_model.pt \
  --output outputs/maven_pipeline_eval_smoke/metrics.json \
  --per-sample-output outputs/maven_pipeline_eval_smoke/predictions.jsonl
```

确认后把 `--limit 0` 跑完整 test。重点查看：

- coarse candidate coverage。
- coarse edge / relation precision、recall、F1。
- refined edge / relation precision、recall、F1。
- refined 相对 coarse 的增量，而不是只看各自 loss。

## 8. MIRAI 本地未来预测

主实验使用冻结外部抽取器预计算的事件文件，4B 只负责最终预测，0.5B adapter 负责粗图。评测器会用 MIRAI 数据集中的 query 和 documents 覆盖外部上下文，只接受事件节点：

```bash
python src/evaluate_local_qwen_pipeline.py \
  --split test \
  --limit 8 \
  --event-source precomputed \
  --precomputed-events datasets/mirai_events_test.jsonl \
  --model-path models/Qwen2.5-4B \
  --coarse-base-model-path models/Qwen2.5-0.5B \
  --coarse-adapter-path outputs/coarse_graph_qwen_lora_4090_full/best_adapter \
  --refinement-model-path outputs/refinement_graph_4090_full/refinement_model.pt \
  --max-docs 4 \
  --max-events 16 \
  --max-pairs 64 \
  --output-dir outputs/local_qwen_pipeline_eval
```

输出：

```text
outputs/local_qwen_pipeline_eval/metrics.json
outputs/local_qwen_pipeline_eval/predictions.jsonl
```

当前入口默认执行 refined graph 路径。论文正式评测前必须补齐 documents-only、events-only、coarse、refined 和 shuffled 图的统一模式。

旧的 Qwen 事件抽取只作为冻结上游基线保留。显式启用：

```bash
python src/evaluate_local_qwen_pipeline.py \
  --split test \
  --limit 8 \
  --event-source qwen \
  --model-path models/Qwen2.5-4B \
  --coarse-base-model-path models/Qwen2.5-0.5B \
  --coarse-adapter-path outputs/coarse_graph_qwen_lora_4090_full/best_adapter \
  --refinement-model-path outputs/refinement_graph_4090_full/refinement_model.pt \
  --output-dir outputs/local_qwen_pipeline_qwen_events
```

该模式不更新 Qwen 抽取参数，也不作为主方法。论文中的所有图对照必须复用同一份预计算事件文件。

## 9. 日志与 checkpoint 原则

1. 控制台只看进度与 epoch 汇总。
2. 样本质量看 `debug_readable.log`。
3. 数值曲线看 `train_history.json`。
4. 复现实验使用 `train_config.json`。
5. 推理默认加载 best，不加载 latest。
6. 评测必须同时保存 aggregate metrics 和 per-sample predictions。
7. 不根据 test 指标调阈值；阈值在 validation 上确定。

## 10. 云端命令注意事项

- Bash 多行命令的反斜杠必须位于行末。
- 不能把参数名拆行，例如必须完整写成 `--negative-ratio 1.5`。
- 启动大任务前先跑 smoke test。
- 训练时使用 `python -u`，确保日志及时刷新。
- 先检查输出目录，避免覆盖已有 best checkpoint。
- 训练结束后同步整个输出目录，而不只同步权重文件。
