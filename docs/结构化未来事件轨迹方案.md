# 结构化未来事件轨迹方案

版本：v0.1
日期：2026-07-22
状态：研究主线方案；后续代码按本文档拆分任务实现

## 1. 一句话定义

本项目的最终目标调整为：

> 让 LLM 在因果图约束下，先生成结构化的未来事件演化轨迹，再完成闭集未来事件预测。

其中因果图不是最终产物，而是用于约束和解释预测过程的中间结构。模型不只输出最终答案，还必须在专门标签中输出若干个 `t` 时刻之前可能发生、并会推动 `t` 时刻事件出现的中间事件。

## 2. 为什么这样做

开放域未来事件预测难评估。若让模型直接生成未来事件，答案可以有多种等价表述，很难判断对错；若只做闭集选择题，评估稳定，但模型可能靠表面相关性猜答案，无法体现因果推理。

因此采用折中任务：

```text
历史文档 + 事件节点 + 因果图 + 候选答案
-> 结构化 forecast_trace
-> 闭集 final_answer
```

闭集 `final_answer` 提供可靠监督，`forecast_trace` 提供可检查、可约束、可用于强化学习的中间预测路径。

## 3. 任务输入输出

### 3.1 输入

正式输入包括：

```json
{
  "query": {
    "subject": "actor_a",
    "object": "actor_b",
    "target_time": "t",
    "task": "predict the relation/event at t"
  },
  "documents": ["cutoff 之前可见的新闻文档"],
  "events": ["严格事件 mention 节点"],
  "causal_graph": {
    "nodes": ["event-input-v1 events"],
    "edges": ["refined causal relation edges"]
  },
  "choices": [
    {"choice_id": "A", "event_code": "010", "description": "..."},
    {"choice_id": "B", "event_code": "180", "description": "..."}
  ]
}
```

事件抽取仍然不作为本项目贡献。MAVEN 使用 gold event mentions，MIRAI 使用冻结外部抽取器或数据集已给出的结构化事件缓存。

### 3.2 输出

预测模型必须输出两个标签：

```text
<forecast_trace>
{
  "intermediate_events": [
    {
      "relative_time": "t-3",
      "event": "opposition groups organize protests",
      "actors": ["opposition groups"],
      "supporting_event_ids": ["e12", "e18"],
      "supporting_edge_ids": ["r7"],
      "expected_effect": "increases likelihood of government crackdown",
      "confidence": 0.71
    },
    {
      "relative_time": "t-1",
      "event": "security forces deploy to the capital",
      "actors": ["security forces"],
      "supporting_event_ids": ["e21"],
      "supporting_edge_ids": ["r11"],
      "expected_effect": "precedes arrests or violent confrontation",
      "confidence": 0.66
    }
  ]
}
</forecast_trace>

<final_answer>
{
  "choice_id": "B",
  "event_code": "180",
  "confidence": 0.74
}
</final_answer>
```

`forecast_trace` 不是普通自然语言思维链，而是结构化的 prospective event trace。它可以被解析、打分、干预和消融。

## 4. 当前项目中应该保留的部分

### 4.1 保留：严格事件输入

保留 `event-input-v1`、事件 mention、trigger、participants、document_id、sentence_index 和 evidence span。新方向更依赖严格事件节点，因为 `forecast_trace` 的 `supporting_event_ids` 必须能回指到真实输入事件。

需要补充：

- 所有事件节点增加可显示的 `normalized_event`。
- 评测日志中统一使用 `trigger + mention + participants`，不再显示整句伪装成事件。
- MIRAI 侧事件缓存必须严格记录 cutoff 和 doc_id。

### 4.2 保留：Qwen 事件对粗图

保留现有 `train_coarse_graph_qwen.py` 和 `run_coarse_graph_qwen.py`。粗图仍然负责从大量候选事件对中提出可能相关的边。

需要调整定位：

```text
旧定位：coarse graph 是 refinement 的输入
新定位：coarse graph 是 forecast_trace 的候选因果支撑来源之一
```

粗图质量不只看 edge F1，还要看它是否能为最终答案提供正确的 supporting path。

### 4.3 保留：graph-level refinement

保留 `refinement_model.py`、`train_refinement.py` 和 `run_refinement.py`。refinement 仍然是两阶段方法的关键模块。

新方向中 refinement 的作用是：

- 删除会误导 forecast_trace 的噪声边。
- 补充粗图漏掉的局部连接。
- 为最终预测提供 frontier node 和高置信支持子图。
- 为 RL 提供可训练的图策略，而不是只做静态图修复。

优先改进不是盲目增大参数，而是加入冻结文本 embedding、真实 coarse prediction 训练分布和预测效用奖励。

### 4.4 保留：本地 Qwen 预测评测

保留 `evaluate_local_qwen_pipeline.py` 作为端到端评测入口。它应扩展为统一支持：

- documents only
- events only
- coarse graph
- refined graph
- shuffled graph
- refined graph + forecast_trace

最终论文主结果必须来自这个端到端评测，而不是只来自 MAVEN 边分类。

### 4.5 保留：RL hook

保留 `rl_pipeline_hooks.py`，但接口要从“只给图打分”扩展到“图 + forecast_trace + final_answer”的 trajectory。

需要新增字段：

```json
{
  "coarse_graph": "...",
  "refined_graph": "...",
  "forecast_trace": "...",
  "final_answer": "...",
  "choice_logprobs": "...",
  "trace_parse_status": "...",
  "reward_breakdown": "..."
}
```

## 5. 需要改变的地方

### 5.1 最终预测 prompt

原先 prompt 只要求模型根据文档和图输出答案。现在必须要求模型先输出结构化标签：

```text
You must first predict a structured future-oriented event trace before the final answer.
The trace must only use visible historical documents, provided event nodes, and the causal graph as support.
Then choose one answer from the closed candidate set.
```

禁止事项：

- 不允许输出候选集合之外的最终答案。
- 不允许在 `forecast_trace` 中引用不存在的 event_id。
- 不允许使用 cutoff 之后的信息。
- 不允许把整句当作事件节点。

### 5.2 评测指标

主指标仍然是闭集未来事件预测：

- Accuracy
- Macro-F1
- Hit@K
- Calibration

新增中间过程指标：

- trace JSON parse rate
- supporting event id validity
- supporting edge id validity
- trace temporal violation rate
- graph support coverage
- trace intervention sensitivity
- removing trace 后 final answer 置信度下降幅度

注意：`forecast_trace` 没有人工 gold，因此不能声称它本身被完全监督正确。它的价值必须通过最终预测提升、图一致性和干预实验来证明。

### 5.3 训练流程

推荐三阶段：

```text
Phase A: supervised graph construction
MAVEN gold events -> coarse scorer -> refinement

Phase B: supervised closed-set forecasting
MIRAI documents/events/graph/choices -> final_answer
可选使用模板或强模型生成 forecast_trace 伪标签

Phase C: RL with structured trace
model generates forecast_trace + final_answer
reward from answer correctness + graph consistency + intervention sensitivity
```

### 5.4 RL 目标

冻结最终预测模型的 base 权重，只训练可控策略：

- coarse LoRA policy
- refinement policy
- 可选 forecast head / small adapter

主奖励：

```text
R_answer = 1 if final closed-set answer is correct else 0
```

辅助奖励：

```text
R_graph_support = trace 中引用的 event/edge 是否存在且与答案候选相关
R_temporal = 中间事件时间是否早于目标 t
R_intervention = 删除 trace 支撑边后，正确答案概率是否下降
R_format = 标签和 JSON 是否可解析
R_density = 图是否过密或引用过多无关边
```

总奖励：

```text
R = w1 * R_answer
  + w2 * R_graph_support
  + w3 * R_temporal
  + w4 * R_intervention
  + w5 * R_format
  - w6 * R_density
```

其中 `R_answer` 权重最高。辅助奖励只用于约束轨迹，不应压过最终任务。

## 6. 论文创新点重写

论文贡献建议集中为三点：

### 6.1 Forecast-trace constrained event prediction

把开放域未来事件预测改成：

```text
structured prospective event trace + closed-set final prediction
```

这样既保留可评估性，又迫使模型在回答前显式构造可检查的未来演化路径。

### 6.2 Coarse-to-refined graph as trace support

不是简单把因果图作为 prompt 附件，而是要求 `forecast_trace` 中的每个中间事件回指到 refined graph 的事件节点和边，使图成为预测路径的结构约束。

### 6.3 RL from answer correctness and graph intervention

不使用全文重构相似度作为主奖励，而是使用闭集答案正确性、图支撑一致性和局部干预敏感性来优化轨迹与图。这样可以减少“文本看起来相似但因果图错误”的奖励投机。

## 7. 实验设计

### 7.1 主实验

在 MIRAI 上比较：

```text
Qwen documents only
Qwen events only
Qwen + coarse graph
Qwen + refined graph
Qwen + refined graph + forecast_trace
Qwen + shuffled graph + forecast_trace
```

如果最后一项也提升，说明模型可能只是在利用格式或额外 token，而不是利用正确图。因此 shuffled graph 是必要负控制。

### 7.2 中间实验

在 MAVEN 上报告：

- candidate recall
- coarse edge F1
- refined edge F1
- refined 相对 coarse 的 delta
- graph density
- cycle rate

这些指标只证明图模块可用，不是论文主结论。

### 7.3 干预实验

对 refined graph 做三种干预：

- 删除 forecast_trace 引用的关键边。
- 反转关键边方向。
- 随机删除同数量非关键边。

观察最终答案概率和选择是否变化。合理结果应是：关键边干预影响更大，非关键边干预影响较小。

## 9. 当前结论

当前项目的大部分代码可以保留。真正需要改变的是最终任务中心：

```text
旧中心：构建更好的因果图，然后看预测是否变好
新中心：让 LLM 生成可检查的未来事件轨迹，因果图负责约束这条轨迹，最终用闭集预测评估
```

这个改法更适合写论文，因为它同时解决三个问题：

- 开放域未来事件预测难评估。
- LLM 直接闭集预测缺少可解释推理过程。
- 因果图如果只看边分类，无法证明对未来预测有用。
